# 📊 SQL Basics – Topic-wise Introduction

This document provides a brief introduction to fundamental SQL concepts covered in this repository. It is designed to help beginners understand *what each concept does* before diving into queries.

---

## 1️⃣ SELECT Basics  
The `SELECT` statement is the foundation of SQL and is used to retrieve data from a database.  
You can fetch all columns using `SELECT *` or specify particular columns as needed.  
Every SQL query begins with SELECT, making it the most essential command to learn.

---

## 2️⃣ WHERE Clause  
The `WHERE` clause is used to filter records based on specific conditions.  
It allows you to extract only relevant data instead of working with the entire dataset.  
Common operators include `=`, `>`, `<`, `BETWEEN`, `IN`, and `LIKE`.

---

## 3️⃣ ORDER BY  
`ORDER BY` is used to sort query results in ascending (`ASC`) or descending (`DESC`) order.  
This helps organize data for better readability and analysis.

---

## 4️⃣ LIMIT / TOP  
These clauses are used to restrict the number of rows returned.  
`LIMIT` is used in MySQL/PostgreSQL, while `TOP` is used in SQL Server.  
This is useful when working with large datasets.

---

## 5️⃣ ALIAS  
Aliases (`AS`) provide temporary names for columns or tables.  
They improve query readability and simplify complex queries.

---

## 6️⃣ Aggregate Functions  
Aggregate functions perform calculations on multiple rows and return a single value.  

Common functions include:
- `COUNT()` → Counts rows  
- `SUM()` → Calculates total  
- `AVG()` → Calculates average  
- `MIN()` → Finds minimum value  
- `MAX()` → Finds maximum value  

These are essential for summarizing data.

---

## 7️⃣ DISTINCT  
`DISTINCT` is used to remove duplicate values from query results.  
It ensures only unique values are returned.

---

## 8️⃣ Basic String Functions  
String functions help manipulate text data.

Common functions:
- `UPPER()` → Converts to uppercase  
- `LOWER()` → Converts to lowercase  
- `LENGTH()` → Finds string length  
- `SUBSTRING()` → Extracts part of a string  
- `CONCAT()` → Combines strings  

These are useful in data cleaning and transformation.

---

## 9️⃣ Comments  
Comments are used to explain SQL queries and improve readability.

- `-- Single-line comment`  
- `/* Multi-line comment */`

---

## 🔟 DDL & DML Commands  

### DDL (Data Definition Language)
Used to define and manage database structure:
- `CREATE`
- `ALTER`
- `DROP`
- `TRUNCATE`

### DML (Data Manipulation Language)
Used to manipulate data within tables:
- `INSERT`
- `UPDATE`
- `DELETE`
- `SELECT`

---

## 💡 Final Note  
SQL is not just about writing queries—it’s about understanding and working with data effectively.  
Mastering these basics builds the foundation for advanced data analysis.
